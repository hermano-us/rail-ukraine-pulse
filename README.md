# Rail Ukraine Pulse

Интерактивная карта публичных статусов пассажирских поездов Укрзалізниці для 22 областей Украины. Карта может работать как на GitHub Pages, так и полностью автономно: статические файлы и API на Cloudflare, непрерывный Playwright-сборщик в Docker.

## Что показывает версия 5

- реальные рейсы из публичной таблицы задержек УЗ;
- стабильный идентификатор рейса: дата события, номер и направление;
- официальный статус, задержку, прогноз и причину отдельно от модельного расчёта;
- положение вдоль железнодорожной геометрии, confidence и погрешность;
- вероятный участок нахождения вместо псевдоточной точки;
- крупные железнодорожные узлы рядом с маршрутом как геометрические ориентиры;
- нормализованный журнал официальных событий;
- реестр источников с состояниями `online`, `stale`, `snapshot`, `candidate`, `protected` и `requires-key`;
- реестр рейсов, поиск, фильтры, сортировку и локальную историю;
- изолированный focus-mode: выбранный поезд, только его маршрут, станции и зона неопределённости.

Демонстрационные поезда не загружаются. `data/trains.json` — исторический артефакт и в runtime не используется.

## Честные ограничения

Публичный источник УЗ предоставляет статусы задерживающихся пассажирских поездов, прогнозы и причины. Он не предоставляет GPS, скорость, полное расписание всех поездов и подтверждённое прохождение каждой станции.

Поэтому:

- оранжевый маркер и выделенный участок являются расчётом, а не телеметрией;
- железнодорожные узлы в карточке подписываются как геометрические ориентиры, а не как факт остановки;
- после 90 минут без новых событий экстраполяция останавливается, а после 180 минут координата скрывается;
- при недоступности источника последние официальные события остаются в реестре, но карта не показывает ложное движение;
- грузовые, специальные и военные перевозки не получают оперативных координат;
- морской слой остаётся без позиций до подключения легального AIS API через серверный прокси.

Публичное вокзальное табло УЗ подключается browser-adapter: backend открывает официальный пользовательский интерфейс, проверяет DOM-контракт и извлекает номер, маршрут, время, путь и задержку. Скрытый гостевой токен не копируется, Cloudflare не обходится. При изменении страницы адаптер переходит в `stale` и сохраняет последний успешный снимок.

## Модель доказательств

Каждая запись приводится к событию:

```json
{
  "runId": "uz:2026-07-17:121/122:миколаїв-пас--київ-пас",
  "type": "forecast_arrival",
  "value": "21:30",
  "occurredAt": "2026-07-17T16:08:13Z",
  "sourceId": "uz-delay-dashboard",
  "authority": "official",
  "confidence": 1
}
```

Официальные события не превращаются в GPS-координаты. Координата создаётся отдельной моделью и всегда содержит метод, confidence, errorKm и время исходного события.

## Расчёт положения

Алгоритм `rail-corridor-v5`:

1. привязывает пункты маршрута к железнодорожному графу;
2. строит путь по геометрии OSM;
3. переводит прогноз УЗ в абсолютное время события;
4. оценивает прогресс по маршруту;
5. рассчитывает confidence и погрешность;
6. преобразует погрешность в участок железнодорожной линии;
7. ищет крупные узлы рядом с этим маршрутом;
8. отдельно хранит время источника и время расчёта;
9. останавливает движение маркера после 90 минут и скрывает координату после 180 минут.

## Источники

Каталог находится в `data/sources.json`.

Подключены backend-адаптерами:

- публичный дашборд задержек УЗ;
- официальное вокзальное табло УЗ через публичный browser-adapter;
- публичная web-лента официального канала УЗ о пригородных поездах;
- Poezdato и Kiyavia как контроль доступности справочных расписаний;
- локальный снимок железнодорожной геометрии OpenStreetMap;
- границы областей geoBoundaries.

Справочные агрегаторы Poezdato и Kiyavia никогда не используются как оперативное подтверждение координаты.

Подготовлены к дальнейшему подключению:

- отчёты АМПУ о судозаходах — нужен адаптер таблиц;
- Wikimedia Commons — фотографии с лицензией и авторством;
- AIS — требуется API-ключ и серверный прокси.

Агрегаторы расписаний никогда не повышают статус позиции до `confirmed`. Для подтверждения прохождения станции требуется официальное событие УЗ или отдельный проверенный источник.

## Частота обновления

- GitHub Actions запускает независимые адаптеры каждые 10 минут;
- карта опрашивает live API каждые 30 секунд;
- локальная история хранит до 48 снимков;
- сбой источника явно отображается как `stale`.

GitHub Actions может запускаться с задержкой, поэтому интерфейс всегда показывает фактический возраст снимка.

## Архитектура

```text
официальные и открытые источники
             │
             ▼
      адаптеры источников
             │
             ▼
  нормализованный журнал событий
             │
             ├── факты и происхождение
             └── модель rail-corridor-v5
                       │
                       ▼
       вероятный участок + confidence
                       │
                       ▼
           Leaflet / реестр / карточка
```

## Автономный production

Карта, API, закрытая админ-диагностика и Docker-сборщик описаны в [docs/AUTONOMOUS_DEPLOYMENT.md](docs/AUTONOMOUS_DEPLOYMENT.md). Для выдачи доступа зрителям используется Cloudflare Access, поэтому приложение не хранит пользовательские пароли.

## Локальный запуск

```bash
npm test
npm run serve
```

Откройте `http://localhost:4173`.

## Публикация

- `.github/workflows/update-data.yml` устанавливает браузерный runtime, запускает тесты и обновляет `live.json` и `source-runtime.json`;
- GitHub Pages публикует ветку `main` после обновления данных;
- секреты AIS нельзя размещать во frontend или JSON репозитория.
