# Rail Ukraine Pulse

Статическое Leaflet-приложение для отображения реальных публичных статусов пассажирских поездов Укрзалізниці в 22 областях Украины. Работает на GitHub Pages без собственного backend.

## Что показывает карта

- только строки, присутствующие в актуальной публичной таблице задержек УЗ;
- статусы «В дорозі» и «Готується», номер, направление, задержку, прогноз и причину;
- расчётное положение поезда вдоль железнодорожного коридора;
- confidence, примерную погрешность, метод и время снимка источника;
- поиск по номеру и маршруту, фильтры областей, состояния и уверенности;
- адаптивную тёмную тему.

**Демонстрационные поезда не загружаются.** Файл `data/trains.json` оставлен только как исторический артефакт и не участвует в runtime.

Публичная страница УЗ является таблицей задержек, а не полным GPS-реестром поездов. Поэтому карта показывает все доступные на ней строки, но не заявляет охват всех поездов Украины. Если для строки нет прогноза прибытия или маршрут не покрыт геометрией, поезд остаётся в оперативной ленте со статусом позиции `unknown` — координата не выдумывается.

Грузовые, специальные и военные поезда не отображаются. Морские объекты остаются пустыми до подключения легального AIS-источника.

## Частота обновления

- GitHub Actions получает новый публичный снимок УЗ каждые **15 минут**;
- открытая страница повторно загружает JSON каждые **60 секунд**;
- при сбое источника сохраняется последний снимок с явным статусом `stale`.

## Модель позиции

УЗ публикует прогноз прибытия, но не GPS, скорость или последнюю пройденную станцию. Для статуса «В дорозі» приложение:

1. привязывает начальную и конечную станции к железнодорожной геометрии;
2. строит путь по графу доступных железнодорожных коридоров;
3. оценивает оставшееся расстояние по прогнозу прибытия и консервативной средней скорости;
4. возвращает `estimated` с низкой/средней уверенностью и погрешностью не менее 25 км.

Метод обозначается как `UZ-arrival-forecast / rail-corridor back-calculation`. Это аналитическая оценка, а не телеметрия.

## Запуск

```bash
npm test
npm run serve
```

Откройте `http://localhost:4173`.

## Обновление и публикация

- `npm run update-data` — обновить публичный снимок;
- `.github/workflows/update-data.yml` — автоматическое обновление каждые 15 минут;
- `.github/workflows/deploy-pages.yml` — публикация GitHub Pages.

API-ключи текущей версии не нужны.
