# Rail Ukraine Pulse

Интерактивная карта реальных публичных статусов пассажирских поездов Укрзалізниці для 22 областей Украины. Frontend работает на GitHub Pages, данные обновляются GitHub Actions.

## Что показывает версия 3

- только рейсы, присутствующие в актуальной публичной таблице задержек УЗ;
- корректную идентичность рейса: дата обслуживания, номер и направление;
- официальный статус, задержку, прогноз и причину отдельно от расчётной координаты;
- оценку положения вдоль железнодорожного графа с confidence и кругом погрешности;
- реестр активных рейсов с поиском, быстрыми фильтрами и сортировкой;
- региональную сводку и диагностику качества источника;
- цепочку доказательств в карточке каждого рейса;
- локальную историю снимков в браузере пользователя.

Демонстрационные поезда не загружаются. `data/trains.json` является историческим артефактом и не участвует в runtime.

## Ограничения источника

Публичная страница УЗ предоставляет официальный статус задержанных пассажирских поездов и прогноз прибытия/отправления. Она не предоставляет:

- GPS-координату;
- текущую скорость;
- последнюю фактически пройденную станцию;
- полный список всех поездов Украины;
- тип конкретного локомотива или состава.

Поэтому оранжевые маркеры являются аналитической оценкой, а не телеметрией. При недостатке данных рейс остаётся в реестре со статусом `unknown` и не получает выдуманную координату.

Грузовые, специальные и военные перевозки не отображаются. Морской слой остаётся пустым до подключения легального AIS-провайдера.

## Модель рейса

```text
runId = provider + serviceDate + trainNumber + direction
```

Пример:

```text
uz:2026-07-17:79/80:львів--дніпро-головний
```

Это предотвращает смешивание встречных направлений и рейсов разных дат.

## Расчёт положения

Алгоритм `rail-graph-v2`:

1. нормализует пункты отправления и назначения;
2. привязывает их к железнодорожному графу;
3. строит путь по доступной геометрии коридоров;
4. переводит прогноз прибытия УЗ в абсолютное время Киева;
5. оценивает оставшееся расстояние по консервативной средней скорости;
6. вычисляет confidence, errorKm и композитное качество данных;
7. визуализирует точку и географическую зону погрешности.

Качество учитывает свежесть источника, наличие прогноза, покрытие маршрута, ошибку привязки и опубликованную надёжность.

## Частота обновления

- GitHub Actions получает новый снимок УЗ каждые **15 минут**;
- открытая карта перечитывает JSON каждые **60 секунд**;
- при сбое используется последний снимок с явным состоянием `stale`;
- локальная история хранит до 48 снимков каждого рейса в браузере.

## Локальный запуск

```bash
npm test
npm run serve
```

Откройте `http://localhost:4173`.

## Публикация

- `.github/workflows/update-data.yml` обновляет публичный снимок;
- `.github/workflows/deploy-pages.yml` запускает тесты и публикует GitHub Pages;
- API-ключи текущей версии не нужны.
