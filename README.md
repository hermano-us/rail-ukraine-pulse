# Rail Ukraine Pulse

Статическое веб-приложение на Leaflet и OpenStreetMap для анализа публичного пассажирского железнодорожного движения по 22 областям Украины. Карта работает на GitHub Pages без собственного backend.

## Что показывает MVP

- 12 демонстрационных публичных пассажирских рейсов, включая состояния «в движении», «на станции», «в депо» и «источник недоступен»;
- выбранные пользователем 22 области с кликабельными границами и фильтрами;
- расчётное положение вдоль геометрии железнодорожного коридора, а не по прямой между городами;
- отдельные статусы `confirmed`, `estimated`, `reported`, `stale`, `unknown`;
- уверенность 0–1, погрешность, метод, источники и последнее подтверждённое событие;
- карточку поезда с названием, маршрутом, описанием, типом состава и лицензированной фотографией;
- историю движения, поиск, фильтры транспорта, состояния, статуса позиции, области и уверенности;
- адаптивную тёмную тему.

Грузовые, специальные и военные поезда намеренно не включены. Точные оперативные координаты не публикуются: демонстрационные положения пассажирских поездов имеют пониженную уверенность и погрешность не менее 8 км.

## Частота обновления

- браузер повторно загружает JSON и пересчитывает положения каждые **60 секунд**;
- GitHub Actions запускает адаптер публичной страницы задержек УЗ каждые **15 минут**;
- при недоступности источника интерфейс честно показывает `unavailable`; последний удачный снимок может использоваться только со статусом `stale`;
- морской слой подготовлен архитектурно, но остаётся пустым до подключения легального публичного AIS-источника.

Данные в `trains.json` — демонстрационные сценарии, а не обещание фактического движения конкретного поезда. Актуальное расписание пользователь проверяет на официальном сайте УЗ.

Слой источников изолирован от клиентского контракта. Новый источник добавляется отдельным адаптером, который записывает нормализованные обновления в `data/live.json`.

## Модель позиции

Алгоритм выбирает соседние события расписания, учитывает фактическое время и задержку, проецирует станции на LineString маршрута и интерполирует накопленное расстояние вдоль линии. Уверенность снижается при длинном временном окне, старом подтверждении, задержке и слабом качестве событий. Для расчётной позиции возвращаются:

```json
{
  "status": "estimated",
  "coordinates": [30.52, 50.45],
  "confidence": 0.73,
  "errorKm": 18,
  "method": "schedule-route-interpolation",
  "lastConfirmedAt": "2026-07-17T10:00:00Z",
  "sources": ["public-schedule", "rail-corridor", "delay-dashboard"]
}
```

## Запуск

Нужен Node.js 20+ только для локального сервера, тестов и обновления JSON. Сборка клиенту не требуется.

```bash
npm run build-data
npm test
npm run serve
```

Откройте `http://localhost:4173`.

## Обновление и публикация

- `npm run update-data` — попытка получить публичную таблицу задержек УЗ;
- `.github/workflows/update-data.yml` — запуск каждые 15 минут и commit только изменившегося снимка;
- `.github/workflows/deploy.yml` — публикация статических файлов на GitHub Pages.

API-ключи в текущем MVP не нужны. Если позже будет выбран AIS-провайдер, секрет добавляется в GitHub Actions Secrets, а не в клиентский JavaScript.


## Гибридный оперативный слой

Версия 2.1 разделяет данные по доказательности:

1. **Официальный публичный статус** — номер, направление, задержка, состояние и прогноз из `uz-vezemo.uz.gov.ua`. Это факт публикации УЗ, но не GPS-координата.
2. **Событие станции** — фактическое прибытие/отправление может считаться подтверждением только при наличии записи официального вокзального табло. Плановая остановка всегда подписывается как модель.
3. **Расчётная позиция** — интерполяция по расписанию и железнодорожному коридору. Она визуально отделена, содержит confidence и погрешность не менее 8 км.
4. **Грузовой слой** — только агрегированная архивная аналитика. Открытые наборы ГИОЦ УЗ последний раз обновлялись в январе–феврале 2022 года; точные номера, live-маршруты и координаты грузовых составов не собираются.

Оперативная лента показывает все полученные публичные статусы, даже если для конкретного рейса пока нет безопасной геометрии маршрута. На карту попадают только рейсы, для которых есть проверенный маршрутный сценарий.

### Источники

- задержки и публичный статус: <https://uz-vezemo.uz.gov.ua/delayform/>;
- официальное вокзальное табло: <https://booking.uz.gov.ua/schedule/>;
- архив пробега грузовых вагонов: <https://data.gov.ua/dataset/ee159558-ff90-4425-b48d-3c1e264b231d>;
- архив грузооборота по станциям отправления и назначения: каталог ГИОЦ УЗ на <https://data.gov.ua/organization/filiia-hiots-pat-ukrzaliznytsia>.

Выбор областей выполняется только чекбоксами слева. Полигоны на карте не интерактивны и не меняют фильтры.
